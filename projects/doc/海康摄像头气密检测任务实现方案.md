# 海康摄像头气密检测任务实现方案

基于 **RoboOS (RoboBrain任务分解) + VLA执行原子任务** + Fourier GR2 机器人的实现方案

## 一、任务概述

### 1.1 任务背景

本任务为海康摄像头产品的气密检测流程自动化，使用傅利叶 GR2 人形机器人（双臂7DOF + 五指灵巧手）完成全部9个操作步骤。

### 1.2 核心设计理念

采用 **"RoboBrain任务分解 + VLA执行原子任务"** 的双层架构：

| 层级 | 模型 | 职责 |
|------|------|------|
| **任务分解层** | RoboBrain 2.0 (Master) | 理解自然语言指令，分解为原子任务序列 |
| **任务执行层** | VLA模型 (Slaver) | 执行具体原子技能，输出机器人控制指令 |

### 1.3 任务流程

| 步骤 | 任务名称 | 描述 |
|------|----------|------|
| 1 | 取摄像头产品 | 左手从周转盒内双指拿取摄像头 |
| 2 | 摘镜头帽+检查 | 右手摘掉镜头帽并放置，目测检查镜头 |
| 3 | 扫描二维码 | 双手转动摄像头，将二维码对准扫码器 |
| 4 | 放置工装 | 镜头朝上水平放入气密检测工装 |
| 5 | 启动检测 | 双手同时按下两个绿色启动按钮 |
| 6 | 取出+检查 | 检测完成后取出摄像头，目测检查 |
| 7 | 盖镜头帽 | 左手取镜头帽盖到摄像头上 |
| 8 | 放入周转盒 | 按顺序整齐放入周转盒 |
| 9 | 查看结果 | 读取屏幕上的气密检测结果 |

### 2.2 VLA端到端控制的优势

| 特性 | 传统方案 (感知→规划→控制) | VLA端到端方案 |
|------|--------------------------|---------------|
| **流程** | 像素→世界坐标→IK→关节角度 | 像素 → 关节角度 (端到端) |
| **中间表示** | bbox, 3D点云, 轨迹点 | 无，直接输出动作 |
| **标定依赖** | 需要手眼标定、相机内参 | 无需标定，从数据学习 |
| **泛化能力** | 依赖标定精度 | 从大规模数据学习泛化 |
| **误差累积** | 多阶段误差累积 | 端到端，无累积 |
| **实时性** | 多次推理 | 单次推理输出动作序列 |

## 四、VLA原子技能执行层

VLA模型采用端到端方式，直接从视觉输入生成机器人关节角度指令，无需中间的坐标转换。

### 4.1 VLA原子技能列表

每个原子技能对应一个训练好的VLA策略，直接输出13-DoF动作序列。

| 技能代码 | 技能名称 | 输入 | 输出 | 说明 |
|----------|----------|------|------|------|
| **VLA_PICK** | 抓取物体 | RGB+Mask+State | action(T,13) | 从指定位置抓取目标物体 |
| **VLA_PLACE** | 放置物体 | RGB+Mask+State | action(T,13) | 将物体放置到指定位置 |
| **VLA_PULL** | 拔出操作 | RGB+Mask+State | action(T,13) | 拔出镜头帽等插入物 |
| **VLA_INSERT** | 插入操作 | RGB+Mask+State | action(T,13) | 插入镜头帽等 |
| **VLA_PRESS** | 按压操作 | RGB+Mask+State | action(T,13) | 按压按钮 |
| **VLA_ROTATE** | 旋转物体 | RGB+Mask+State | action(T,13) | 双手协同旋转物体 |
| **VLA_HANDOVER** | 双手传递 | RGB+Mask+State | action(T,13) | 左右手物体传递 |
| **VLA_INSPECT** | 视觉检测 | RGB | 缺陷描述 | 外观缺陷检测(VLM) |
| **VLA_READ** | 屏幕读取 | RGB | 文本内容 | OCR读取结果(VLM) |


## 五、各步骤VLA技能执行

每个步骤由RoboBrain分解为子任务，然后调用对应的VLA原子技能，**VLA直接输出13-DoF/26-DoF关节角度动作序列**。

## 六、关键技术要点

### 6.1 力控精度 (端到end VLA实现)

GR2 灵巧手需实现亚牛顿级力控（0.5-2N）避免损伤镜头和镜头帽。

**端到end VLA实现方式**：
```python
# ✅ VLA从演示数据学习合适的手部力控
# VLA输出的action[7:13]自然包含力控信息
action_sequence = vla_controller.predict(...)  # (T, 13)

for action in action_sequence:
    # 直接执行,无需手动调整Kp/Kd
    client.set_joint_positions({
        "right_manipulator": action[0:7],
        "right_hand": action[7:13]  # VLA已学习合适的闭合程度
    })
```

**与传统方法对比**:
```python
# ❌ 传统方法:需要手动整定Kp/Kd参数
kp = max(10, 50 - target_force * 20)  # 经验公式,不稳定
kd = 1.0
client.set_motor_cfg(...)
```

**VLA优势**:
- 从演示数据学习不同物体的合适抓取力
- 无需针对每个物体手动调参
- 泛化到相似物体

### 6.2 对准精度

镜头帽插拔需达到 **±0.3mm** 重复定位精度。

**端到end VLA实现方式**：
- 使用视觉伺服闭环 (第10.5节)
- VLA多次推理逐步逼近目标
- 从演示数据学习精确插入动作

```python

await vla_visual_servo_loop(
    skill_id="VLA_INSERT",
    target="镜头帽到镜头",
    tolerance_mm=0.3
)
```

**精度来源**:
1. 演示数据中的高精度标注
2. Diffusion Policy强大的泛化能力
3. 视觉伺服多次迭代修正

### 6.3 双手协调

M2 技能需支持主从手同步，时间差 **<100ms**。

**端到end VLA实现方式**：
```python
# VLA Diffusion Policy自然编码时间依赖
action_sequence = vla_controller.predict(skill_id="VLA_PRESS")  # (T, 26)

# 左右手动作天然同步,时间差<10ms
for action in action_sequence:
    client.set_joint_positions({
        "left_manipulator": action[0:7],    # 左臂
        "left_hand": action[7:13],          # 左手
        "right_manipulator": action[13:20], # 右臂 (同一时刻)
        "right_hand": action[20:26]         # 右手
    })  # 单次API调用,同步发送
    await asyncio.sleep(0.01)
```

**VLA如何学习同步**:
- 演示数据中左右手动作序列时间对齐
- Diffusion Policy输出单个时间步的26-DoF动作
- 无需额外同步逻辑

### 6.4 视觉引导

所有抓取操作融合多相机实时观测。

**端到end VLA调用**：

## 十、端到端VLA原子技能实现方案

本章详细描述如何使用 **DexGraspVLA式端到端模型** 实现感知、移动、抓取、操作等原子技能。VLA直接从视觉输入输出关节角度序列，无需中间坐标转换。

### 10.1 端到端VLA架构

```
┌─────────────────────────────────────────────────────────────────┐
│                   多相机视觉输入                                  │
├─────────────────────────────────────────────────────────────────┤
│  头部深度相机(第三视角) + 双臂腕部相机(第一视角) + 机器人状态     │
│  RGB: (480,640,3) + Depth: (480,640,1) + State: (13,)           │
└─────────────────────┬───────────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────────┐
│                VLA端到端策略网络 (Diffusion Policy)              │
├─────────────────────────────────────────────────────────────────┤
│  • 图像编码器 (ResNet/ViT)                                       │
│  • 状态编码器 (MLP)                                              │
│  • 扩散去噪网络 (U-Net)                                          │
│  • 输出: action_sequence (T, 13) 或 (T, 26)                     │
└─────────────────────┬───────────────────────────────────────────┘
                      │ 直接输出关节角度!
                      ▼
┌─────────────────────────────────────────────────────────────────┐
│              Fourier GR2 机器人关节控制                           │
├─────────────────────────────────────────────────────────────────┤
│  client.set_joint_positions({                                   │
│      "right_manipulator": action[0:7],   # 手臂7-DoF            │
│      "right_hand": action[7:13]          # 灵巧手6-DoF          │
│  })                                                             │
└─────────────────────────────────────────────────────────────────┘
```

**核心优势**:
- ✅ 无需手眼标定、相机内参、坐标转换
- ✅ 无需逆运动学(IK)计算
- ✅ 端到端学习,从演示数据直接获得精度
- ✅ 单次推理输出完整动作序列

### 10.2 VLA原子技能列表

每个技能都是独立训练的端到端策略,直接输出关节角度序列。

| 技能代码 | 技能名称 | 输入 | 输出 | DoF | 说明 |
|----------|----------|------|------|-----|------|
| **VLA_PICK** | 抓取物体 | RGB+Depth+Mask+State | action(T,13) | 13 | 单臂抓取 |
| **VLA_PLACE** | 放置物体 | RGB+Depth+Mask+State | action(T,13) | 13 | 单臂放置 |
| **VLA_PULL** | 拔出操作 | RGB+Depth+Mask+State | action(T,26) | 26 | 双臂拔出镜头帽 |
| **VLA_INSERT** | 插入操作 | RGB+Depth+Mask+State | action(T,26) | 26 | 双臂插入镜头帽 |
| **VLA_PRESS** | 按压操作 | RGB+Depth+Mask+State | action(T,26) | 26 | 双手同时按按钮 |
| **VLA_ROTATE** | 旋转物体 | RGB+Depth+Mask+State | action(T,26) | 26 | 双手协同旋转 |
| **VLA_HANDOVER** | 双手传递 | RGB+Depth+Mask+State | action(T,26) | 26 | 左右手物体传递 |

### 10.3 多相机数据流管理

#### 相机布局
```yaml
cameras:
  head_camera:           # 头部深度相机 - 第三视角
    type: "realsense_d435"
    resolution: [640, 480]
    modality: ["rgb", "depth"]
    use_for: ["VLA_PICK", "VLA_PLACE", "VLA_ROTATE", "VLM_INSPECT"]

  left_wrist_camera:     # 左臂腕部相机 - 第一视角
    type: "usb_rgb"
    resolution: [640, 480]
    modality: ["rgb"]
    use_for: ["VLA_INSERT", "VLA_PULL"] # 精细操作

  right_wrist_camera:    # 右臂腕部相机 - 第一视角
    type: "usb_rgb"
    resolution: [640, 480]
    modality: ["rgb"]
    use_for: ["VLA_INSERT", "VLA_PULL"] # 精细操作
```

#### 相机选择策略

```python
class CameraManager:
    """管理多相机输入,为不同技能选择合适的相机"""

    def __init__(self):
        self.cameras = {
            "head": RealSenseCamera("/dev/video0"),
            "left_wrist": USBCamera("/dev/video2"),
            "right_wrist": USBCamera("/dev/video4")
        }

    def get_observation(self, skill_name: str, active_arm: str = None):
        """根据技能选择相机组合"""

        if skill_name in ["VLA_PICK", "VLA_PLACE", "VLA_ROTATE", "VLA_PRESS"]:
            # 粗操作: 使用第三视角 + 主动手臂腕部相机
            return {
                "rgb_third_view": self.cameras["head"].get_rgb(),
                "depth": self.cameras["head"].get_depth(),
                "rgb_wrist": self.cameras[f"{active_arm}_wrist"].get_rgb()
            }

        elif skill_name in ["VLA_INSERT", "VLA_PULL"]:
            # 精细操作: 双臂腕部相机 + 头部深度
            return {
                "rgb_third_view": self.cameras["head"].get_rgb(),
                "depth": self.cameras["head"].get_depth(),
                "rgb_left_wrist": self.cameras["left_wrist"].get_rgb(),
                "rgb_right_wrist": self.cameras["right_wrist"].get_rgb()
            }

        elif skill_name in ["VLM_INSPECT", "VLM_READ"]:
            # 纯视觉任务: 只需要RGB
            return {
                "rgb": self.cameras["head"].get_rgb()
            }

        else:
            raise ValueError(f"Unknown skill: {skill_name}")
```

### 10.4 端到端VLA技能实现代码

```python
# hikvision_endtoend_skills.py - 端到端VLA技能库 (无坐标转换)

import asyncio
import numpy as np
from mcp.server.fastmcp import FastMCP
from fourier_aurora_client import AuroraClient
from inference import UnifiedInference  # RoboBrain VLM (仅用于检测/OCR)
import torch

mcp = FastMCP("hikvision_endtoend", stateless_http=True, host="0.0.0.0", port=8001)

# ==================== 模型初始化 ====================

# VLA端到端策略模型 (DexGraspVLA式)
from dexgraspvla_controller import DexGraspVLAController

vla_controller = DexGraspVLAController(
    checkpoint="/path/to/hikvision_vla_checkpoint",
    device="cuda:0"
)

# RoboBrain VLM (仅用于外观检测和OCR,不输出动作)
vlm_model = UnifiedInference("BAAI/RoboBrain2.0-7B")

# SAM分割模型
from segment_anything import sam_model_registry, SamPredictor
sam = sam_model_registry["vit_h"](checkpoint="/path/to/sam_vit_h.pth")
sam_predictor = SamPredictor(sam)
sam_predictor.model.to("cuda:0")

# 相机管理器
camera_mgr = CameraManager()

# 机器人客户端
client = AuroraClient.get_instance(domain_id=123)

# ==================== 目标分割 ====================

def segment_target(rgb_image: np.ndarray, prompt: str) -> np.ndarray:
    """使用SAM分割目标物体,返回二值Mask"""
    sam_predictor.set_image(rgb_image)

    # 简化: 使用点提示 (实际可结合VLM定位中心点)
    # 这里假设目标在图像中心附近
    h, w = rgb_image.shape[:2]
    input_point = np.array([[w//2, h//2]])
    input_label = np.array([1])

    masks, scores, _ = sam_predictor.predict(
        point_coords=input_point,
        point_labels=input_label,
        multimask_output=True,
    )

    # 返回得分最高的mask
    best_mask = masks[np.argmax(scores)]
    return best_mask.astype(np.float32)

# ==================== 单臂/双臂模式管理 ====================

class RobotStateManager:
    """管理机器人状态和单臂/双臂模式切换"""

    def __init__(self):
        self.left_state = "idle"   # idle | busy | holding
        self.right_state = "idle"
        self.held_objects = {"left": None, "right": None}
        self.mode = "single_arm"   # single_arm | bimanual

    def can_execute(self, skill_name: str, arm: str = None):
        """检查是否可以执行技能"""
        if skill_name in ["VLA_ROTATE", "VLA_PRESS", "VLA_PULL", "VLA_INSERT"]:
            # 双臂技能
            return self.left_state == "idle" or self.right_state == "idle"
        else:
            # 单臂技能
            if arm == "left":
                return self.left_state == "idle"
            elif arm == "right":
                return self.right_state == "idle"
        return False

    def set_busy(self, arm: str):
        if arm == "left":
            self.left_state = "busy"
        elif arm == "right":
            self.right_state = "busy"
        elif arm == "both":
            self.left_state = "busy"
            self.right_state = "busy"
            self.mode = "bimanual"

    def set_idle(self, arm: str):
        if arm == "left":
            self.left_state = "idle"
        elif arm == "right":
            self.right_state = "idle"
        elif arm == "both":
            self.left_state = "idle"
            self.right_state = "idle"
            self.mode = "single_arm"

    def get_current_state(self, arm: str) -> np.ndarray:
        """获取当前机器人状态 (13-DoF或26-DoF)"""
        if arm == "left":
            joint_pos = client.get_joint_positions(["left_manipulator", "left_hand"])
            return np.concatenate([
                joint_pos["left_manipulator"],  # 7-DoF
                joint_pos["left_hand"]           # 6-DoF
            ])  # (13,)

        elif arm == "right":
            joint_pos = client.get_joint_positions(["right_manipulator", "right_hand"])
            return np.concatenate([
                joint_pos["right_manipulator"],
                joint_pos["right_hand"]
            ])  # (13,)

        elif arm == "both":
            joint_pos = client.get_joint_positions([
                "left_manipulator", "left_hand",
                "right_manipulator", "right_hand"
            ])
            return np.concatenate([
                joint_pos["left_manipulator"],   # 7-DoF
                joint_pos["left_hand"],          # 6-DoF
                joint_pos["right_manipulator"],  # 7-DoF
                joint_pos["right_hand"]          # 6-DoF
            ])  # (26,)

@mcp.tool()
async def VLA_PICK(target: str, arm: str = "left") -> dict:
    """【VLA端到端】抓取物体 - 直接输出关节角度序列

    Args:
        target: 抓取目标描述 (如"摄像头"、"镜头帽")
        arm: 使用哪只手臂 ("left" | "right")

    Returns:
        {"success": bool, "final_state": np.ndarray(13,)}
    """
    # 1. 检查状态
    if not robot_state_mgr.can_execute("VLA_PICK", arm):
        return {"success": False, "error": f"{arm} arm is busy"}

    robot_state_mgr.set_busy(arm)

    # 2. 获取多相机观测
    obs = camera_mgr.get_observation("VLA_PICK", active_arm=arm)
    rgb_third = obs["rgb_third_view"]   # (480, 640, 3)
    depth = obs["depth"]                 # (480, 640)
    rgb_wrist = obs["rgb_wrist"]         # (480, 640, 3)

    # 3. SAM分割目标
    mask = segment_target(rgb_third, target)  # (480, 640)

    # 4. 获取当前机器人状态
    current_state = robot_state_mgr.get_current_state(arm)  # (13,)

    # 5. VLA推理 - 端到端输出关节角度序列!
    action_sequence = vla_controller.predict(
        rgb_third_view=rgb_third,
        mask=mask[..., None],  # (480, 640, 1)
        depth=depth[..., None],
        rgb_wrist=rgb_wrist,
        robot_state=current_state,
        skill_id="VLA_PICK",
        language_instruction=f"抓取{target}"
    )  # 输出: (T, 13) - 直接是关节角度!

    # 6. 直接执行动作序列 (无需IK/坐标转换)
    for t, action in enumerate(action_sequence):
        client.set_joint_positions({
            f"{arm}_manipulator": action[0:7].tolist(),
            f"{arm}_hand": action[7:13].tolist()
        })
        await asyncio.sleep(0.01)  # 100Hz控制频率

    # 7. 更新状态
    robot_state_mgr.held_objects[arm] = target
    robot_state_mgr.set_idle(arm)

    return {
        "success": True,
        "target": target,
        "arm": arm,
        "action_steps": len(action_sequence),
        "final_state": action_sequence[-1].tolist()
    }
```

### 10.5 端到端VLA视觉伺服闭环

对于高精度操作(如镜头帽插入±0.3mm),VLA模型通过视觉伺服闭环逐步逼近目标。

```python
async def vla_visual_servo_loop(skill_id: str, target: str, tolerance_mm: float = 0.3):
    """VLA视觉伺服闭环 - 无需坐标转换

    通过多次VLA推理+执行,逐步提高精度

    Args:
        skill_id: VLA技能ID (如"VLA_INSERT")
        target: 目标描述
        tolerance_mm: 容许误差(mm),通过图像像素尺度估算

    Returns:
        {"success": bool, "iterations": int, "final_error_mm": float}
    """
    max_iterations = 10
    pixel_to_mm_scale = 0.1  # 简化:假设1像素≈0.1mm (实际从VLA训练数据学习)

    for i in range(max_iterations):
        # 1. 获取当前观测
        obs = camera_mgr.get_observation(skill_id)
        current_state = robot_state_mgr.get_current_state("both")

        # 2. SAM分割目标
        mask = segment_target(obs["rgb_third_view"], target)

        # 3. VLA推理 - 输出微调动作序列
        action_sequence = vla_controller.predict(
            rgb_third_view=obs["rgb_third_view"],
            mask=mask[..., None],
            depth=obs["depth"][..., None],
            rgb_left_wrist=obs["rgb_left_wrist"],
            rgb_right_wrist=obs["rgb_right_wrist"],
            robot_state=current_state,
            skill_id=skill_id,
            language_instruction=f"精细对准{target}",
            num_inference_steps=5  # 少量步数,微调
        )  # (5, 26)

        # 4. 执行微调动作
        for action in action_sequence:
            client.set_joint_positions({
                "left_manipulator": action[0:7].tolist(),
                "left_hand": action[7:13].tolist(),
                "right_manipulator": action[13:20].tolist(),
                "right_hand": action[20:26].tolist()
            })
            await asyncio.sleep(0.01)

        # 5. 估算误差 (简化:通过mask中心偏移量)
        mask_center = np.array([
            np.mean(np.where(mask)[1]),  # x
            np.mean(np.where(mask)[0])   # y
        ])
        image_center = np.array([320, 240])
        pixel_error = np.linalg.norm(mask_center - image_center)
        error_mm = pixel_error * pixel_to_mm_scale

        print(f"[VLA Servo] 迭代{i+1}: 像素偏差={pixel_error:.1f}px, 估算误差={error_mm:.2f}mm")

        if error_mm < tolerance_mm:
            print(f"[VLA Servo] ✓ 对准完成,精度: {error_mm:.3f}mm")
            return {"success": True, "iterations": i+1, "final_error_mm": error_mm}

        await asyncio.sleep(0.1)  # 等待稳定

    return {"success": False, "iterations": max_iterations, "final_error_mm": error_mm}
```

**与传统闭环的区别**:
- ❌ 传统:需要像素→世界坐标→IK,多次转换累积误差
- ✅ VLA:直接从图像推理微调关节角度,无中间转换

**闭环集成点** (在需要高精度的步骤中调用):
```python
# 在 hikvision_step8_replace_cap() 中启用闭环
async def hikvision_step8_replace_cap() -> dict:
    await VLA_PICK(target="检测台上的镜头帽", arm="left")

    # ⚠️ 高精度插入,启用视觉伺服闭环
    servo_result = await vla_visual_servo_loop(
        skill_id="VLA_INSERT",
        target="镜头帽到镜头",
        tolerance_mm=0.3
    )

    if not servo_result["success"]:
        print("[ERROR] 镜头帽插入精度不足!")
        return {"cap_replaced": False, "error": "alignment_failed"}

    # 执行最终插入动作
    await VLA_INSERT(target="镜头帽到镜头")
    return {"cap_replaced": True, "iterations": servo_result["iterations"]}
```

### 10.8 工程化注意事项

#### 10.8.1 VLA推理延迟与控制频率

**问题**: VLA推理300ms vs 100Hz控制频率无法共存

**解决方案**:
```python
# 预测性流水线: VLA推理与动作执行overlap
async def execute_with_pipeline(skill_id, obs):
    # VLA推理 (300ms) → 生成T=30步动作序列
    action_seq = vla_controller.predict(obs, skill_id)  # (30, 13)

    # 执行30步 (30 * 10ms = 300ms) 同时VLA推理下一段
    for i, action in enumerate(action_seq):
        # 执行当前动作
        client.set_joint_positions(action)

        # 最后5步时开始推理下一段
        if i == 25:
            next_obs = camera_mgr.get_observation()
            next_task = asyncio.create_task(
                vla_controller.predict(next_obs, skill_id)
            )

        await asyncio.sleep(0.01)  # 100Hz

    # 获取下一段动作序列
    next_action_seq = await next_task
    return next_action_seq
```

**实际控制频率**:
- VLA推理频率: ~3Hz (每300ms一次)
- 关节控制频率: 100Hz (通过插值/重复动作实现)

#### 10.8.2 相机标定简化方案

**端到端VLA的优势**: 无需精确手眼标定,从数据学习隐式映射

**仍需标定的场景**:
- 视觉伺服闭环: 需要像素→mm的粗略尺度
- 多相机融合: 需要相对位姿(简化标定)

**简化标定流程**:


#### 10.8.3 失败恢复机制

```python
class SkillExecutor:
    async def execute_with_retry(self, skill_fn, max_retries=3):
        for attempt in range(max_retries):
            try:
                result = await skill_fn()
                if result["success"]:
                    return result

                # VLA执行失败,重新采集观测
                print(f"[Retry {attempt+1}] 技能执行失败,重试...")
                await asyncio.sleep(0.5)

            except Exception as e:
                print(f"[Error] {e}")
                if attempt == max_retries - 1:
                    # 最终失败,回到安全姿态
                    await self.return_to_home_position()
                    raise

        return {"success": False, "error": "max_retries_exceeded"}
```
